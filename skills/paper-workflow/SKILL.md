# Paper Workflow — 论文写作工作流

## When to Use

- 开始新论文项目时
- 需要规范化论文写作流程时
- 人类要求参考工作流时

## How It Works

### 流程

```
/paper-start → 人类提供研究主题
    ↓
/paper-plan → 我规划结构，人类审查
    ↓
/lit-review → 搜索/整理文献，保存到 papers/
    ↓
人类确认研究方法（我不会的空着标注）
    ↓
/data-analysis → 代码执行计算，输出结果
    ↓
人类审查结果，确认结论方向
    ↓
各章节写作 → 人类逐章审查
    ↓
/paper-review → 质量审查
    ↓
/paper-export → 导出最终论文
```

### 关键节点

1. **研究主题** — 人类决定，我只建议
2. **论文结构** — 我规划，人类审查确认
3. **文献管理** — 搜索/下载/整理，保存到本地
4. **研究方法** — 人类确认，不会的空着标注
5. **数据分析** — 代码执行，输出代码+结果
6. **结论方向** — 人类确认，我只提供草稿
7. **论文写作** — 逐章写作，人类逐章审查
8. **质量审查** — 逻辑/引用/数据/格式
9. **导出** — LaTeX/Word

### 每一步都需要人类确认

- 我不会自动跳到下一步
- 人类可以随时打回重来
- 全流程自动只有在人类明确要求时才启用

## Examples

```
用户: /paper-start 股票市场波动性研究

我: 确认研究主题：股票市场波动性研究
    创建项目目录：output/paper-stock-volatility/
    初始化文献索引：papers/references.json
    
    请确认：
    1. 研究问题是什么？
    2. 研究假设是什么？
    3. 需要搜索哪些文献？
```
