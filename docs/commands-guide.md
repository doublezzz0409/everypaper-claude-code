# everypaper-claude-code 命令说明书

## 概述

everypaper-claude-code 提供 6 个斜杠命令，覆盖论文写作的完整生命周期。

核心理念：**"你负责研究方向，我负责工具操作"** — 每一步都需要人类确认，AI 不会自动推进下一步。

---

## 命令总览

| 命令 | 用途 | 触发时机 |
|------|------|----------|
| `/paper-start` | 开始新论文 | 开始一个新研究项目时 |
| `/paper-plan` | 论文规划 | 确定主题后，规划论文结构时 |
| `/data-analysis` | 数据分析 | 需要执行数值计算时 |
| `/lit-review` | 文献综述 | 搜索、整理、保存文献时 |
| `/paper-review` | 论文审查 | 写完章节后，审查质量时 |
| `/paper-export` | 导出论文 | 所有章节审查通过后，导出最终版本时 |

---

## 命令详解

### 1. `/paper-start` — 开始新论文

初始化一个论文写作项目。

**流程：**
1. 确认研究主题（人类提供）
2. 创建 `output/paper-{topic}/` 项目目录
3. 初始化 `papers/references.json`（空文献索引）
4. 调用 `paper-planner` 代理规划结构
5. 输出大纲，等待人类确认后才能继续

**参数：** `$ARGUMENTS` — 研究主题描述（可选，也可交互式确认）

**输出：**
- `output/paper-{topic}/` — 项目目录
- `output/paper-{topic}/outline.md` — 论文大纲（待人类确认）
- `papers/references.json` — 文献索引（空）

**注意：**
- 研究方向由人类决定
- 大纲需要人类审查确认后才能继续
- 不会自动开始写作

---

### 2. `/paper-plan` — 论文规划

规划论文结构，输出大纲和待确认事项。

**流程：**
1. 读取已有的研究主题和文献
2. 调用 `paper-planner` 代理
3. 输出论文结构规划
4. 标注所有待人类确认的事项
5. 等待人类审查

**参数：** `$ARGUMENTS` — 可选：指定已有论文项目目录

**输出：**
- 论文结构大纲
- 待确认事项清单（方法、变量、数据来源）
- 建议（基于学术规范）

**注意：**
- 研究方向由人类决定
- 分析方法不确定时标注 `[待人类确认]`
- 人类确认后才能继续下一步

---

### 3. `/data-analysis` — 数据分析

执行数据分析，强制使用代码，输出代码和结果。

**流程：**
1. 确认数据来源（来自哪篇文献、哪个数据集）
2. 确认计算方法（人类已确认的公式/模型）
3. 如果公式不确定 → 输出 `[待人类确认]` 标注，停止
4. 编写 Python/R 代码执行计算
5. 输出代码 + 结果
6. 调用 `data-verifier` 代理验证

**参数：** `$ARGUMENTS` — 分析描述（如 "计算描述性统计"、"回归分析"）

**输出：**
- `output/data/analysis.py` — 计算代码
- `output/data/results.txt` — 计算结果
- 数据来源标注
- 审查清单

**铁律：**
- 所有计算必须用代码 — 禁止心算、估算、编造数字
- 输出代码 + 结果 — 人类需要看到完整的计算过程
- 不会的公式 → 标注空位 — 绝不编造公式
- 数据来源必须标注 — 每个数字都要有出处

**注意：**
- 公式不确定时停止，等待人类确认
- 数据来源必须标注
- 代码必须可运行、可审查

---

### 4. `/lit-review` — 文献综述

搜索、整理、保存文献到本地 `papers/` 目录。

**流程：**
1. 确认搜索关键词（人类提供或建议）
2. 确认搜索范围（年份、期刊、领域）
3. 执行搜索，返回结果列表
4. 人类确认哪些需要下载
5. 下载文献 PDF，保存到 `papers/`
6. 更新 `papers/references.json`

**参数：** `$ARGUMENTS` — 搜索关键词（可选，也可交互式确认）

**输出：**
- `papers/` — 下载的文献 PDF
- `papers/references.json` — 文献元数据索引
- 搜索结果摘要（待人类确认相关性）

**注意：**
- 搜索前需要人类确认关键词
- 下载后需要人类确认相关性
- 所有文献必须保存到本地，方便复查
- 记录来源 URL/DOI

---

### 5. `/paper-review` — 论文质量审查

审查论文的逻辑、引用、数据、格式。

**流程：**
1. 读取论文内容
2. 调用 `paper-reviewer` 代理
3. 调用 `truth-reviewer` 代理
4. 输出审查报告
5. 标注需要修复的问题

**参数：** `$ARGUMENTS` — 可选：指定审查范围（如 "只审查引言"）

**审查维度：**

| 维度 | 检查项 |
|------|--------|
| 逻辑审查 | 研究问题是否清晰？假设有理论支撑？方法匹配研究问题？结论由数据支撑？ |
| 引用审查 | 每个引用可在 `papers/` 找到原文？引用格式一致？遗漏引用？ |
| 数据审查 | 每个数字有来源？计算有代码记录？结果合理？ |
| 格式审查 | 章节结构完整？图表编号连续？参考文献格式统一？ |

**输出：**
- 审查报告（CRITICAL / HIGH / MEDIUM / LOW）
- 需要修复的问题清单
- 打回重来的理由（如有）

---

### 6. `/paper-export` — 导出论文

将论文导出为 LaTeX 或 Word 格式。

**流程：**
1. 读取所有章节内容
2. 整合引用列表
3. 生成 LaTeX 或 Word 文件
4. 输出到 `output/final/`

**参数：** `$ARGUMENTS` — 导出格式（`latex` 或 `word`，默认 `latex`）

**输出：**
- `output/final/paper.tex` — LaTeX 文件
- `output/final/paper.docx` — Word 文件（如果选择）
- `output/final/references.bib` — 参考文献

**注意：**
- 导出前需要人类确认所有章节已审查通过
- 引用格式需要人类确认
- 不会自动修改内容

## 命令关系图

```
/paper-start  →  /paper-plan  →  /lit-review  →  /data-analysis
     │              │               │                │
  创建项目       规划结构        搜集文献         分析数据
     │              │               │                │
  人类确认       人类确认        人类确认         人类确认
                                                    │
                                            /paper-review  →  /paper-export
                                                 │                │
                                              审查质量          导出论文
                                                 │                │
                                              人类确认         人类确认
```

## 设计原则

1. **人类在环** — 每一步都需要人类确认，AI 不会自动推进
2. **禁止编造** — 不确定的内容标注 `[待人类确认]`，绝不编造
3. **代码执行** — 所有计算必须用代码，禁止心算
4. **来源可查** — 每个引用必须在 `papers/` 有原文，每个数字必须有出处
5. **可审查性** — 所有代码、数据、中间结果必须可追溯
